## 1. FOUNDER DOCTRINE & CONSTITUTIONAL PRINCIPLES (DNA)

AI TO AI HOLDING is not an AI Agent, a Chatbot, a SaaS platform, a software application, or a simple multi-agent script. It is an **AI-Native Corporation (Digital Organization)**. We do not build software first; we mirror real-world corporate governance, legal risk management, financial auditiability, and evolutionary learning loops into a continuous digital operating system.

### Core Philosophy
1. **Organization First, Product Second:** We construct an autonomous digital corporate structure. Any revenue-producing entity (such as Customs Intelligence) is merely an interchangeable revenue unit or division inside this parent framework.
2. **We Build the OS, AI is the Workforce:** The entire corporation runs via specialized AI operational units. However, the ultimate governing authority, override privileges, and foundational constitutional updates remain strictly with the human Chairman.
3. **Real World First Principle:** We do not design from clean-room software abstractions. We look at how physical multi-national corporations operate, survive, and get audited, and then we translate those real-world checks and balances into code.
4. **Organization Before Automation:** Never automate a process before defining which Corporate Office owns that process, what the exact role responsibility is, and how that action will be scrutinized.

### Constitutional Laws (Non-Negotiable Baseline)
* **Law 1: Ultimate Human Sovereignty.** No AI unit possesses the authority to override a human command, remove the Chairman, alter the baseline Constitution, or disable the systemic Audit logs.
* **Law 2: Separation of Knowledge & Agency.** No specialized Agent or Division owns data. Data and context are centralized property managed solely by the *Knowledge Office*. Agents are stateless execution vectors that request temporary knowledge context.
* **Law 3: Mandatory Evidence-Based Output.** AI units are strictly forbidden from guessing, hallucinating, or answering queries without attaching verifiable source references and explicit legal disclaimers. 
* **Law 4: Absolute Traceability.** Every single input query, internal thinking loop, state transition, decision outcome, API token consumption, and financial cost metric must be explicitly written into immutable audit ledgers.
* **Law 5: Evolutionary Governance (Governance Must Be Earned).** The corporation does not generate hundreds of arbitrary operational rules upfront. Instead, the operational layer grows organically: 
    $$\text{Real Experience} \rightarrow \text{Anomalies Identified} \rightarrow \text{AI CEO Analysis} \rightarrow \text{Chairman Review} \rightarrow \text{New Operational Rule Policy Deployment}$$

## 2. REAL-WORLD BUSINESS REALITY CONSTRAINTS
The architecture must compile and run flawlessly inside the following "Ground Truth" operational constraints specified by the Chairman:
* **Deployment Infrastructure:** Strictly Local On-Premise (Standard PC / Consumer Hardware). Architecture must prioritize low-overhead, lightweight, file-based systems. **SQLite** is designated as the core relational database management system. No heavy, multi-node cloud-native microservices allowed during testing.
* **Human-In-The-Loop Availability:** Minimal to None at inception. The system must run autonomously without gating operations on real-time human micro-approvals. Instead, it must heavily rely on automated validation logs, anomaly detection algorithms, and batch reviews.
* **Financial & Computational Footprint:** Hyper-optimized. Minimal token recursion and guardrails against infinite multi-agent ping-pong loops to preserve local computing resources and API spend.
* **Public Readiness Goal:** Infinite internal optimization loop. The system will not be launched publicly until the structural baseline, learning models, and organizational data loops prove 100% stable in the sandbox environment.

#### D. Treasury Office (The Cost Accountant)
* **Role Directive:** Measures financial performance. Tracks computational overhead (API tokens expended, context window utilization, local runtime execution time) against processing revenue to calculate micro-margins for every single transaction.

## 5. REVERSE-ENGINEERED UNIFIED DATABASE SCHEMA (SQLite Specification)

To prevent architectural drifting, technical debt, or organizational alignment failure, the entire database schema is consolidated into a single unified execution script. All keys are explicitly linked to build a rock-solid, fully auditable digital corporate backbone.

```sql
-- =====================================================================
-- AI TO AI HOLDING - COMPLETE UNIFIED SYSTEM SCHEMA SPECIFICATION
-- Target DBMS: SQLite (Local, Lightweight, Highly Optimized File-Based)
-- =====================================================================

PRAGMA foreign_keys = ON;

-- ---------------------------------------------------------------------
-- PART A: CORE INFRASTRUCTURE & ORGANIZATIONAL KNOWLEDGE REGISTRIES
-- ---------------------------------------------------------------------

-- 1. Office Registry: Identifies core corporate office pillars
CREATE TABLE corporate_offices (
    office_id TEXT PRIMARY KEY,
    office_name TEXT NOT NULL UNIQUE,
    responsibility_description TEXT NOT NULL,
    is_active INTEGER DEFAULT 1 CHECK (is_active IN (0, 1)),
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 2. Division Registry: Identifies revenue-producing components 
CREATE TABLE organizational_divisions (
    division_id TEXT PRIMARY KEY,
    division_name TEXT NOT NULL UNIQUE,
    office_owner_id TEXT NOT NULL,
    revenue_model_type TEXT NOT NULL, -- 'KaaS', 'AaaS', 'Subscription'
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (office_owner_id) REFERENCES corporate_offices(office_id)
);

-- 3. Corporate Agents Register: Track identities of autonomous execution units
CREATE TABLE corporate_agents (
    agent_id TEXT PRIMARY KEY,
    agent_name TEXT NOT NULL,
    assigned_division_id TEXT NOT NULL,
    assigned_role_profile TEXT NOT NULL,
    current_status TEXT NOT NULL DEFAULT 'IDLE', -- 'IDLE', 'PROCESSING', 'MAINTENANCE'
    system_prompt_version TEXT NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (assigned_division_id) REFERENCES organizational_divisions(division_id)
);

-- 4. Document Repository: Master storage references for internalized corporate literature
CREATE TABLE document_repository (
    document_id TEXT PRIMARY KEY,
    document_title TEXT NOT NULL,
    document_type TEXT NOT NULL, -- 'LEGAL_MANUAL', 'TARIFF_SCHEDULE', 'REGULATION_TEXT'
    raw_content TEXT NOT NULL,
    source_origin_url TEXT,
    cryptographic_hash TEXT NOT NULL UNIQUE,
    uploaded_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 5. Knowledge Registry: Structured factual data blocks extracted by Knowledge Office
CREATE TABLE knowledge_registry (
    knowledge_id TEXT PRIMARY KEY,
    associated_document_id TEXT NOT NULL,
    domain_tags TEXT NOT NULL, -- JSON or comma-separated string e.g., 'Customs, HS_Code, Thailand'
    key_entity TEXT NOT NULL, -- e.g., 'HS_85171300'
    factual_assertion_data TEXT NOT NULL, -- The extracted ground-truth truth parameters
    confidence_score REAL NOT NULL CHECK (confidence_score BETWEEN 0.0 AND 1.0),
    last_verified_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (associated_document_id) REFERENCES document_repository(document_id)
);

-- 6. Knowledge Graph Relations: Semantic structural linking between data primitives
CREATE TABLE knowledge_graph_edges (
    edge_id TEXT PRIMARY KEY,
    source_knowledge_id TEXT NOT NULL,
    target_knowledge_id TEXT NOT NULL,
    relationship_type TEXT NOT NULL, -- 'SUB_CATEGORY_OF', 'SUPERSEDES', 'CROSS_REFERENCE'
    weight REAL DEFAULT 1.0,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (source_knowledge_id) REFERENCES knowledge_registry(knowledge_id),
    FOREIGN KEY (target_knowledge_id) REFERENCES knowledge_registry(knowledge_id)
);

-- ---------------------------------------------------------------------
-- PART B: THE OPERATIONAL REVENUE DOMAIN (CUSTOMS CORE)
-- ---------------------------------------------------------------------

-- 7. Document Intake Queue: Inbound commercial data records processing pipeline
CREATE TABLE customs_intake_queue (
    intake_id TEXT PRIMARY KEY,
    client_identifier TEXT NOT NULL,
    payload_type TEXT NOT NULL, -- 'COMMERCIAL_INVOICE', 'PACKING_LIST', 'BILL_OF_LADING'
    raw_payload_data TEXT NOT NULL,
    processing_status TEXT DEFAULT 'PENDING' CHECK (processing_status IN ('PENDING', 'PROCESSING', 'COMPLETED', 'FAILED')),
    received_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 8. HS Classification Cache & Reference Lookup Ledger
CREATE TABLE hs_classification_ledger (
    classification_id TEXT PRIMARY KEY,
    intake_id TEXT NOT NULL,
    declared_product_description TEXT NOT NULL,
    predicted_hs_code TEXT NOT NULL,
    rationale_text TEXT NOT NULL,
    confidence_rating REAL NOT NULL,
    applied_knowledge_id TEXT NOT NULL,
    classified_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (intake_id) REFERENCES customs_intake_queue(intake_id),
    FOREIGN KEY (applied_knowledge_id) REFERENCES knowledge_registry(knowledge_id)
);

-- 9. Duty Estimation Engine Logs
CREATE TABLE duty_estimation_records (
    estimation_id TEXT PRIMARY KEY,
    classification_id TEXT NOT NULL,
    assessable_value_amount REAL NOT NULL,
    currency_code TEXT NOT NULL DEFAULT 'THB',
    applicable_duty_rate REAL NOT NULL,
    calculated_duty_payable REAL NOT NULL,
    regulatory_exception_flags TEXT, -- JSON string detailing exemptions applied
    calculated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (classification_id) REFERENCES hs_classification_ledger(classification_id)
);

-- ---------------------------------------------------------------------
-- PART C: COMPLETE TRACEABILITY, FINANCIAL AUDITS & EVOLUTIONARY LEARNING
-- ---------------------------------------------------------------------

-- 10. Audit Event Log: Low-level system telemetry interceptor
CREATE TABLE audit_event_logs (
    event_id INTEGER PRIMARY KEY AUTOINCREMENT,
    timestamp DATETIME DEFAULT CURRENT_TIMESTAMP,
    originating_component TEXT NOT NULL, -- e.g., 'AGENT_HS_CLASSIFIER', 'KNOWLEDGE_OFFICE'
    execution_severity TEXT NOT NULL, -- 'INFO', 'WARN', 'ERROR', 'CRITICAL'
    trace_message_payload TEXT NOT NULL,
    stack_trace_data TEXT
);

-- 11. Decision Log: High-level auditable discrete business outputs
CREATE TABLE corporate_decision_records (
    decision_id TEXT PRIMARY KEY,
    transaction_context_id TEXT NOT NULL, -- Maps back to intake_id or job_id
    acting_agent_id TEXT NOT NULL,
    input_parameters_json TEXT NOT NULL,
    reasoning_process_steps TEXT NOT NULL, -- Step-by-step thinking track
    final_output_json TEXT NOT NULL,
    governance_compliance_status TEXT NOT NULL, -- 'PASSED', 'BLOCKED', 'PENDING_HUMAN_REVIEW'
    executed_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (acting_agent_id) REFERENCES corporate_agents(agent_id)
);

-- 12. Treasury Financial Cost Tracker: Records exact micro-margins for compute calls
CREATE TABLE treasury_cost_ledger (
    transaction_id INTEGER PRIMARY KEY AUTOINCREMENT,
    associated_decision_id TEXT NOT NULL,
    inbound_tokens_count INTEGER NOT NULL,
    outbound_tokens_count INTEGER NOT NULL,
    raw_api_cost_usd REAL NOT NULL,
    allocated_local_compute_ms INTEGER NOT NULL,
    estimated_total_cost_thb REAL NOT NULL,
    revenue_generated_thb REAL DEFAULT 0.0,
    net_profit_margin_thb REAL NOT NULL,
    logged_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (associated_decision_id) REFERENCES corporate_decision_records(decision_id)
);

-- 13. System Governance Operational Policy Layers (Constitutional Rules vs Operational Guidelines)
CREATE TABLE governance_policy_rules (
    rule_id TEXT PRIMARY KEY,
    policy_layer TEXT NOT NULL CHECK (policy_layer IN ('CONSTITUTIONAL', 'OPERATIONAL')),
    rule_name TEXT NOT NULL UNIQUE,
    rule_logic_expression TEXT NOT NULL, -- Clear rules expression or prompt directive
    is_enforced INTEGER DEFAULT 1 CHECK (is_enforced IN (0, 1)),
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 14. Organizational Issues Log (The Shamma Meeting Registry / Evolutionary Feed)
CREATE TABLE organizational_learning_issues (
    issue_id INTEGER PRIMARY KEY AUTOINCREMENT,
    source_decision_id TEXT NOT NULL, -- Tracks down the exact concrete failure event
    root_cause_classification TEXT NOT NULL, -- 'MISSING_KNOWLEDGE', 'PROMPT_ERROR', 'WORKFLOW_FLAW', 'HALO_EFFECT'
    detailed_anomaly_analysis TEXT NOT NULL,
    proposed_policy_remedy TEXT, -- AI CEO written draft proposal for Chairman review
    chairman_approval_status TEXT DEFAULT 'PENDING_REVIEW' CHECK (chairman_approval_status IN ('PENDING_REVIEW', 'APPROVED', 'REJECTED')),
    applied_governance_rule_id TEXT, -- Rule created as a direct result of resolution
    resolved_at DATETIME,
    FOREIGN KEY (source_decision_id) REFERENCES corporate_decision_records(decision_id),
    FOREIGN KEY (applied_governance_rule_id) REFERENCES governance_policy_rules(rule_id)
);
